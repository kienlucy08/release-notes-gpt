- Bug: Legs count on site visit page always shows 4
Strategic Theme: Improve UX/UI
Area of Impact: Site Visit Page
Description: The Site Visit page incorrectly displays all lattice towers as having 4 legs, even when the actual leg count is different in the data attached to the site visit.
Details:
- The displayed leg count doesn't match the structure_legs_count value in the P&T payload attached to the site visit.
- Steps to reproduce include viewing a site visit, checking the structure_legs_count in the JSON data, and observing the discrepancy.
Impact to Users:
- Technicians will now see the correct number of legs displayed on the Site Visit screen, improving accuracy during quality control checks.

- Bug: Visual bug on underlines on tabs
Strategic Theme: Improve UX/UI
Area of Impact: Dashboard
Description: Visual issues with underlines on tabs in the dashboard causing curly underlines and gaps in selected versus all items.
Details:
- Main tabs will now have straight line underlines for better visual alignment.
- Sub tabs on the surveys tab will be fully filled in when selected.
Impact to Users:
- Users will experience a cleaner and more consistent visual design in the dashboard, making navigation easier and more intuitive.

- Feature: Default report names
Strategic Theme: Improve UX/UI
Area of Impact: Report Generation
Description: Default report names will be automatically generated according to a set pattern, improving quality of life for QC Platform users.
Details:
- Report names will follow the pattern: <SITE ID>_TIA Inspection Report_<Creation Timestamp>.
- Creation Timestamp will be human-readable and reflect the report generation time.
- Users can still customize the report names during generation.
Impact to Users:
- QC writers will save time with automatically generated report names, following FieldSync's standard reporting workflow.
- Users can easily customize report names to meet specific customer requirements.
