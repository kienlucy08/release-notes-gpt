Bug: Legs Count Incorrectly Displayed on Site Visit Page
Strategic Theme: Improve UX/UI
Area of Impact: Site Visit Page
Description: The Site Visit page always shows lattice towers as having 4 legs, regardless of the actual leg count stored in the P&T payload.
Details:
- Despite the structure_legs_count value in the P&T payload, the Site Visit page displays "Legs | 4" for all lattice towers.
- This discrepancy leads to inaccurate information being shown to users.
Fix:
- Update the Site Visit page to display the correct number of legs based on the structure_legs_count value.
Impact to Users:
- Users will now see the accurate number of legs for lattice towers on the Site Visit page.
- Provides correct information for quality control checks.
