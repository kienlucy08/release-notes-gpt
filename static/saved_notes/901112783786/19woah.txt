**Bug: "Your Request Rate is Too High" Error in Bedrock Agent**  
*Strategic Theme:* System Hardening  
*Area of Impact:* Bedrock Model Invocation  
When using the Multi-Agentic RAG Query feature, an error stating "Your request rate is too high" occurs in the deployed web application. This error is related to high request rates and can be fixed by implementing throttling at both the global API level and individual API request endpoints.

*Details:*  
- Error message appears due to high request rates in the deployed web application
- Possible causes include low request rates for the current agent model and multiple requests to the Bedrock manager agent
- Solution involves implementing throttling to control the number of model invocation requests

*Impact to Users:*  
- Users will not encounter the error message related to high request rates
- System performance will be optimized with controlled request rates  
(Note: This fix may involve complex backend configurations not directly relevant to general users.)

**Enhancement: QC Editor Scan Image Selector: Arrow Key Navigation and Selection Distinction**  
Improve UX/UI  
In the Scan Image Selector, users can now navigate through images using arrow keys without automatically selecting them. They can explicitly choose which images to include, allowing for multiple non-consecutive selections. Visual cues indicate the currently viewed image and those selected for inclusion.
*Details:*  
• Arrow keys now navigate through images without selecting them  
• Added explicit selection action for users to mark images for inclusion  
• Visual distinction between currently viewed and selected images  
*Impact to Users:*  
• Easier review of multiple images before selecting  
• Ability to select multiple non-consecutive images

*Details:*

**Feature: Scan Inferences: Automatic Queuing for Object Detection on Upload**
